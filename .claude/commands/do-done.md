# ЗАВЕРШЕНИЕ

1. **Анализ изменений**:
   ```bash
   git status
   git diff --stat
   git branch --show-current
   ```

   - Проанализируй измененные файлы
   - Определи суть изменений (что именно сделано)
   - Сформируй краткое описание (1-2 слова): например "добавлен logout", "исправлен баг авторизации"

2. **Коммит** - если есть незакоммиченные изменения:
   ```bash
   git add .
   git commit -m "feat: {описание из п.1}"
   ```

3. **Текущая ветка** - запомни название feature ветки

4. **Merge в main**:
   ```bash
   git checkout main
   git pull origin main 2>/dev/null || true
   git merge {feature-branch} --no-ff -m "feat: {описание из п.1}"
   ```

   **Если конфликты** → останови, сообщи пользователю

5. **Удали feature ветку**:
   ```bash
   git branch -d {feature-branch}
   ```

6. **История** - добавь запись в `.claude/ddrspecs/HISTORY.md` (в начало после разделителя `---`):
   ```markdown
   ## {дата YYYY-MM-DD} - {описание из п.1}

   **Решение**: {краткое описание подхода/библиотеки если использовалась}
   **Файлы**: {основные измененные файлы}

   ```

7. **Финальный коммит**:
   ```bash
   git add .claude/ddrspecs/HISTORY.md
   git commit -m "chore: обновлена история задач"
   ```

**Вывод**: "✅ Смержено в main, история обновлена"
