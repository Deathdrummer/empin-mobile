## Важные решения и подходы

### Архитектурные решения
- **Action Sheet вместо custom popup**: Использование нативного Action Sheet обеспечивает нативный UX и лучшую производительность
- **expo-image-picker**: Официальная библиотека Expo с автоматической обработкой permissions и кросс-платформенностью
- **Позиционирование кнопок внизу**: `bottom: 5` вместо центрирования для корректного отображения при многострочном тексте

### Best Practices
- Запрос permissions непосредственно перед использованием функции
- Проверка статуса permissions с показом понятных сообщений об ошибках
- allowsEditing: true для возможности кадрирования медиа
- quality: 0.8 для баланса между качеством и размером файла

### Выбранные библиотеки
- `@expo/react-native-action-sheet`: Обновлена 6 декабря 2025, активно поддерживается
- `expo-image-picker`: Интегрирована с Expo SDK 54, стабильная версия

## Проблемы и нерешенные вопросы

### Текущие ограничения
1. **Нет серверной интеграции**: Медиа выбирается, но не отправляется на сервер (TODO комментарии в коде на строках 208 и 256)
2. **Нет валидации файлов**: Отсутствует проверка:
   - Размера файла (например, максимум 10MB для фото, 50MB для видео)
   - Формата файла
   - Соотношения сторон видео
3. **Нет обработки множественного выбора**: Сейчас можно выбрать только 1 файл за раз

### Вопросы для обсуждения
- Нужна ли поддержка множественного выбора медиа (несколько фото в одном комментарии)?
- Какие ограничения по размеру файлов установить?
- Нужно ли сжатие видео на клиенте перед отправкой?
- Где хранить загруженные медиа на сервере?
