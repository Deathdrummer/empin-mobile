# Task 1: Модифицировать BottomMenu с условными пропсами

## Цель
Добавить пропсы showCalendar и showFilter в компонент BottomMenu для условного отображения кнопок "Календарь" и "Фильтр".

## Контекст
BottomMenu сейчас всегда показывает все 4 кнопки. Нужно сделать кнопки "Календарь" и "Фильтр" опциональными через пропсы, при этом сохранить визуальную структуру из 4 позиций (пустые слоты вместо скрытых кнопок).

**Best practice**: Условный рендеринг через пропсы - стандартный паттерн React для переиспользуемых компонентов.

## Файлы
- `src/components/BottomMenu.js`

## План
1. [ ] Прочитать текущую реализацию BottomMenu.js
2. [ ] Добавить пропсы showCalendar и showFilter (по умолчанию true)
3. [ ] Обернуть кнопки "Календарь" и "Фильтр" в условный рендеринг
4. [ ] Сохранить структуру из 4 позиций (пустые View вместо скрытых кнопок)
5. [ ] Убедиться что пропсы hasActiveFilters и onClearFilters игнорируются если showFilter=false

## Критерии готовности
- [ ] Пропсы showCalendar и showFilter добавлены с default значением true
- [ ] При showCalendar=false кнопка "Календарь" не отображается, но позиция сохраняется
- [ ] При showFilter=false кнопка "Фильтр" не отображается, но позиция сохраняется
- [ ] Кнопка "Аккаунт" всегда на 4-й позиции справа
- [ ] Существующее использование в Timesheet работает без изменений
