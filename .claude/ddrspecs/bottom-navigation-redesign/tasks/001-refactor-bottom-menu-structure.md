# Task 1: Рефакторинг структуры BottomMenu

## Цель
Переработать компонент BottomMenu для поддержки новой концепции навигации с фиксированными крайними кнопками и динамическими средними.

## Контекст
Текущий BottomMenu использует условные рендеры через `showCalendar`/`showFilter`. Нужно изменить подход:
- Позиция 0 (левая): кнопка "три точки" - **фиксированная для всех разделов**
- Позиции 1-2 (средние): динамические кнопки в зависимости от раздела
- Позиция 3 (правая): кнопка переключения раздела - **фиксированная для всех разделов**

## Проблемы и решения

### 🐛 Проблема 1: Неправильная ориентация иконок
**Дата**: 2026-01-19 17:45
**Описание**:
- Левая кнопка использовала горизонтальные точки (`ellipsis-horizontal`), а должны быть вертикальные
- Правая кнопка имела фиксированную иконку переключения, а должна меняться в зависимости от активного раздела

**Решение**:
- Изменена иконка левой кнопки с `ellipsis-horizontal` на `ellipsis-vertical`
- Добавлена динамическая логика для правой кнопки:
  - `section === 'timesheet'` → показывает `chatbubbles-outline` (иконка мессенджера)
  - `section === 'messenger'` → показывает `calendar-outline` (иконка план-графика)

**Изменено**:
- `src/components/BottomMenu.js:22` - иконка левой кнопки
- `src/components/BottomMenu.js:28-39` - логика правой кнопки с динамической иконкой
