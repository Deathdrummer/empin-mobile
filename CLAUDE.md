## Архитектурные особенности

### Таблицы users и staff — разные ID
В AsyncStorage хранится объект `user` со следующими полями:
- `user.id` — ID из таблицы `users` (аутентификация)
- `user.staff_id` — ID из таблицы `staff` (профиль сотрудника)

API эндпоинты (getAllStaff и др.) возвращают ID из таблицы `staff`.

**Правило:** при сравнении текущего пользователя с данными из staff-запросов — всегда использовать `user.staff_id`, а не `user.id`.
