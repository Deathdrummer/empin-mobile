# Task 1: Модифицировать BottomMenu с условными пропсами

## Цель
Добавить пропсы showCalendar и showFilter в компонент BottomMenu для условного отображения кнопок "Календарь" и "Фильтр".

## Контекст
BottomMenu сейчас всегда показывает все 4 кнопки. Нужно сделать кнопки "Календарь" и "Фильтр" опциональными через пропсы, при этом сохранить визуальную структуру из 4 позиций (пустые слоты вместо скрытых кнопок).

**Best practice**: Условный рендеринг через пропсы - стандартный паттерн React для переиспользуемых компонентов.

## Файлы
- `src/components/BottomMenu.js`

## План
1. [x] Прочитать текущую реализацию BottomMenu.js
2. [x] Добавить пропсы showCalendar и showFilter (по умолчанию true)
3. [x] Обернуть кнопки "Календарь" и "Фильтр" в условный рендеринг
4. [x] Сохранить структуру из 4 позиций (пустые View вместо скрытых кнопок)
5. [x] Убедиться что пропсы hasActiveFilters и onClearFilters игнорируются если showFilter=false

## Критерии готовности
- [x] Пропсы showCalendar и showFilter добавлены с default значением true
- [x] При showCalendar=false кнопка "Календарь" не отображается, но позиция сохраняется
- [x] При showFilter=false кнопка "Фильтр" не отображается, но позиция сохраняется
- [x] Кнопка "Аккаунт" всегда на 4-й позиции справа
- [x] Существующее использование в Timesheet работает без изменений
