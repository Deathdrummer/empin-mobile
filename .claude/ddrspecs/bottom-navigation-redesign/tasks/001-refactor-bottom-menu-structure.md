# Task 1: Рефакторинг структуры BottomMenu

## Цель
Переработать компонент BottomMenu для поддержки новой концепции навигации с фиксированными крайними кнопками и динамическими средними.

## Контекст
Текущий BottomMenu использует условные рендеры через `showCalendar`/`showFilter`. Нужно изменить подход:
- Позиция 0 (левая): кнопка "три точки" - **фиксированная для всех разделов**
- Позиции 1-2 (средние): динамические кнопки в зависимости от раздела
- Позиция 3 (правая): кнопка переключения раздела - **фиксированная для всех разделов**

## Файлы
- `src/components/BottomMenu.js`

## План
1. [ ] Изменить структуру пропсов: убрать `showCalendar`/`showFilter`, добавить `section` или аналогичный проп
2. [ ] Создать renderLeftButton() - кнопка "три точки" (заглушка на этом этапе)
3. [ ] Создать renderRightButton() - кнопка переключения раздела (заглушка на этом этапе)
4. [ ] Создать renderMiddleButtons() - условный рендер средних кнопок по разделу
5. [ ] Обновить layout компонента: 4 позиции с правильными рендерами
6. [ ] Удалить устаревшую логику с handleAccountPress (будет заменена на меню "три точки")

## Критерии готовности
- [ ] BottomMenu принимает новые пропсы вместо старых
- [ ] Структура из 4 позиций с заглушками крайних кнопок
- [ ] renderMiddleButtons() корректно переключается между разделами
- [ ] Код компилируется без ошибок
